SVN

关于SYN

  • Subversion 一个自由开源的版本控制系统

  • 版本控制系统历史 CVS -> SVN（集中式） -> Git（分布式）

  • C / S 架构

  • Apache

Window下的SVN部署

  • 服务端 + 客户端：VisualSVN + TortoiseSVN
  • VisualSVN -> VisualSVN Server Menager -> Create New Repository
  • TortoiseSVN -> 重启 -> 右键菜单
    • improt文件夹
    • New 文件夹 -> SVN Checkout（克隆 Server 的仓库，需要Server库的URL）-> 变动（add文件） -> Commit（上传）
    • update（更新）
    • 回滚 -> update to revesion

版本冲突

​ 若从低版本跨版本commit ，和多用户一起提交同一个文件，使用edit去修改
