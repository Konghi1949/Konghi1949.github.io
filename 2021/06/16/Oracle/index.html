几种删除指令区别

  • delete 用于删除记录但不能释放Oracle空间，只是把原来占用的空间标记为unused

  • truncate 可直接删除大量记录并可释放空间，但不可恢复

  • drop 将删除表的结构被依赖的约束constrain 触发器trigger 索引index，依赖于该表的存储过程/函数将保留,但是变为invalid状态

操作(DML)

  • select

    • 算术运算：+ - * / ->列名->as |name| ，别名

      ​ 加含有null的字段是，用nvl(column_name,0)代替

    • 关系运算：> < <= |<> != ^=|

    • 逻辑运算：and or not

    • 字符串连接操作符：column1|| ‘->’ ||column2

    • 高级查询：

      • distinct 去重
      • NULL操作
      • in操作
      • between ..and..
      • 范围查询
      • like模糊查询：%表示0-n个任意字符，_一个占位字符 ，‘S%’s开头，‘__S’
      • 集合运算查询：intersect交集/union并集/union all可重复并集/minus补集
      • union结果集合并

    • ```sql
      ->select distinct |column_name| from |table_name|; –去重
      ->select * from |table_name| 

      where |column_name| is [not] NULL;    --查找NULL列

      ->select * from |table_name| 

      where |column_name| in (value,...,value);    --in匹配等于集合

      ->select * from |table_name| 

      where |column_name| 
    between |min_value| and |max_value|;    --between操作

      ->select * from |table_name| 

      where |column_table| 
    like '语句';    --模糊查询

      ->select …

      |intersect/union/union all/minus| 
    select...;    --集合运算

      ->select |value|,…,|value| from dual; –常量查询,dual维表
      ->insert into |teble| –结果集批量插入

      ->select |value|,...,|value| from dual
    union
        select |value|,...,|value| from dual
            ...;    --并结果集

    • 子查询

      • 位置：where 条件
                from 子查询的结果集作为外部查询的临时表
      • 结果：单行 0.1(><=)，多行 0.1.n 行(any all some,in,not,in)

    • 伪列

      • rowid：返回该行的物理地址，快速定位记录

      • rownum：返回行号 1 - N(结果集内)，限制select结果 ( >= N) 的显示条数

      • 分页查询：限制返回的区间，前提是把rownum的列放在临时表并为它别名以在方便调用，并为该临时表别名，达成条件（原理：rownum依赖于结果集，放在结果集里产生临时表后的rownum就从伪列转换成了实列，调用时就十分灵活，而为什么必须别名，是为了临时表里的实列rownum和查询表的伪列rownum不起冲突，临时表的别名是为了方便调用实列rownum）

    • 分组 group by |column|…having… 配合聚合函数用于统计 +having过滤
